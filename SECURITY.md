## 报告安全漏洞

我们非常重视安全问题。如果你发现了安全漏洞，请按照以下方式报告：

### 报告方式

**请不要**在公开的 GitHub Issues 中报告安全漏洞。

请发送邮件至：[xqian7024@gmail.com](mailto:xqian7024@gmail.com)

邮件主题格式：`[Security] koa-next-app - 简短描述`

### 报告内容

请尽可能包含以下信息：

1. **漏洞描述**：清晰地描述漏洞类型和影响
2. **复现步骤**：详细的复现说明
3. **影响范围**：哪些版本受影响
4. **可能的修复建议**（可选）
5. **你的联系方式**：以便我们跟进

### 响应时间

- **确认收到**：24 小时内
- **初步评估**：3-5 个工作日内
- **修复计划**：评估后尽快提供

## 安全实践

### 项目已实施的安全措施

- ✅ JWT 认证，支持 Token 刷新
- ✅ 密码使用 bcrypt 加密存储
- ✅ 原生 SQL 使用参数化查询，防止 SQL 注入
- ✅ CORS 配置
- ✅ Helmet 安全头部
- ✅ 输入数据使用 Zod 验证

### 部署安全建议

1. **环境变量**
   - 使用强密码和密钥（至少 32 字符）
   - 不要将 `.env` 文件提交到版本控制
   - 生产环境使用密钥管理服务

2. **数据库**
   - 使用专用数据库用户，限制权限
   - 启用 SSL 连接
   - 定期备份

3. **Docker**
   - 定期更新基础镜像
   - 运行容器时使用非 root 用户
   - 限制容器资源

4. **网络**
   - 使用 HTTPS
   - 配置适当的防火墙规则
   - 定期更新依赖
